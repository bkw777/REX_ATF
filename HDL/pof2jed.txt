
minimal:
$ pof2jed rexbrd.pof -verbose -device 1504as

all the options we might want, still to be tested:
pof2jed rexbrd.pof -verbose -device 1504as -pinclk -pin_keep -MC_power on -power_reset -GCLK3_ITD -JTAG on -TDI_PULLUP -TMS_PULLUP


-pinclk        for ALE ? don't know what this exactly does

-pin_keep      for CSA & CSB ?
               we want CSA & CSB to internally hold up if possible
               unknown yet: does this drive the pin or merely pullup?
               unknown yet: will this cause all other pins to fight the bus?
               If we can't replicate the xilinx internal pullup, not the end of
               the world. We really only need these 2 pins to be pulled up,
               and there is plenty of pcb space for 2 resistors.

-MC_power on   definitely want the chip to sleep as much as possible
               but make sure it doesn't make it too slow

-power_reset   large vcc hysteresis, maybe we want it, maybe we don't, test both

-GCLK3_ITD     Prevents the chip partially waking up if the line changes.
               Probably has no effect in our case.
               Pin 35 is only an output from our point of view so we're always
               driving it one way or another and nothing else is, it's only
               connected the /CE input on the flash chip.

-JTAG on       should already be the default but can't hurt

-TDI_PULLUP    not strictly necessary, but the jtag pins are floating inputs
-TMS_PULLUP

-slew          who knows... try all

-secure        DO NOT USE - This will make the chip unprogrammable any more.
               VPP does NOT overcome this!
